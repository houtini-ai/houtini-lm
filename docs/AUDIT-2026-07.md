# Post-refactor audit (July 2026)

A fresh adversarial review of `main` after the hardening + `node:sqlite` +
inference-lock work, weighted toward the newly-written code. Four independent
finders (node:sqlite migration, inference lock, `index.ts` session changes,
security/regression), findings verified against the source. Everything below was
**fixed in the same pass**; pre-existing/tracked items are noted at the end.

## Fixed

### Inference lock (`inference-lock.ts`) — mutual-exclusion holes
The first cut's cross-process test only exercised live-holder contention, not the
dead-holder steal path, which hid real bugs:
- **Non-atomic steal** — two waiters could both unlink a dead lock and both
  create → two concurrent holders. **Fixed:** conditional steal (re-read and only
  unlink if the on-disk token is unchanged) + per-acquisition token.
- **Unconditional unlink** in `release()`/exit handler deleted *another* process's
  lock after a steal, cascading extra holders. **Fixed:** every unlink is gated on
  the on-disk token still being ours — a process can never delete a lock it
  doesn't own.
- **PID reuse** made the age fallback unreachable (a reused PID pinned a dead
  lock for the full wait). **Fixed:** staleness is age-first, then dead-PID.
- **Keepalive gated on `progressToken`** — a client that omits it got a silent
  multi-minute wait and timed out. **Fixed:** without a token the wait fails open
  at 45s (under the typical client timeout) instead of sitting silent.

*Verified:* 6 processes racing to steal a dead-PID lock → 6 acquires, **zero
overlapping critical sections**; an old-but-live-PID lock stolen in ~1ms.
*Residual (documented):* a sub-millisecond simultaneous-steal window can briefly
double-acquire; it's self-healing and never cascades. Acceptable for a
fail-open optimisation.

### `node:sqlite` migration (`model-cache.ts`)
- **Hard crash on Node 22.5–22.12** — `node:sqlite` needs `--experimental-sqlite`
  there, which the plain shebang doesn't pass, so a static import crashed the
  server on a "supported" engine. **Fixed:** the import is now type-only + a
  guarded dynamic import; if `node:sqlite` is unavailable the cache is disabled
  (server runs without persistence) and every `initDb()` consumer guards on null.
- **Corrupt-DB recovery** renamed only the main file, leaving `-wal`/`-shm`
  sidecars that SQLite replays into the fresh DB; and the corruption `try/catch`
  wrapped only `openDb()`, not the schema DDL. **Fixed:** quarantine the sidecars
  too, and wrap open + schema in the corruption handler.

### `index.ts`
- **File-confinement broke under symlinked roots** — `readGuardedFile` compared
  `realpath(file)` against roots that were only `resolve()`'d, so a root under a
  symlinked ancestor (macOS `/tmp`→`/private/tmp`) failed *every* confined read.
  **Fixed:** roots are `realpathSync`'d at startup.
- **`</think>`-strip heuristic was lose-lose** — the backtick guard leaked
  reasoning that contained code (the primary code-analysis workload) yet still
  deleted answers with an unquoted literal `</think>`. **Fixed:** strip the orphan
  closer only for models whose cached profile says they emit think blocks;
  otherwise leave it (a visible leak beats silent answer deletion).

### Security
- **HF model-card injection** — a squatted card's `license` was interpolated
  verbatim into `discover`/`list_models` output. **Fixed:** `sanitizeCardText`
  collapses control chars/newlines and caps length before storage.

## Cleared (finders checked, no defect)
`p()` coercion, `node:sqlite` bigint handling, the `recordPerformance` upsert
arithmetic, the `initDb` singleton, the `buildSystemPrompt` refactor (no dropped
behaviour), `computeTokPerSec`, the SSE error/flush/throw logic, `redactUrl`
coverage, and read-after-write correctness under the removed snapshot machinery.

## Pre-existing / tracked (not regressions from this session)
- **Forgeable footer trust boundary** (#20) — model output relayed with the same
  `---` delimiter the footer uses; real fix is MCP structured content.
- **`code_task_files` confinement is opt-in** — `HOUTINI_LM_FILE_ROOTS` unset =
  no directory restriction (size cap still applies). By design for backward
  compatibility; document prominently.
- **Residual TOCTOU** in `readGuardedFile` when roots are set (dir component
  swapped between check and read) — requires write access inside a root; low
  severity.
- **`getReasoningEffortValue`** returns `'low'` for unknown backends (#22) — a
  behaviour tradeoff (dropping it also drops the valid value for DeepSeek/OpenAI).
